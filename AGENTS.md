# Open Platform

This repo is docs-first. Start with `docs/README.md`.

Use repo-managed QMD via `mise` for document discovery when the right file is unclear, then read the returned files before making claims or edits.

Use the `docs-grounding` skill for documentation-heavy questions. If docs and implementation differ, report the drift explicitly.

## Git Workflow

Use `Graphite` for all normal git operations in this repo.

- create and manage stacked branches with `Graphite`
- keep pull requests pragmatically small
- keep changes independently reviewable and independently testable where practical
- do not use `main` for normal in-flight feature work
